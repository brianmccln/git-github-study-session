Git / GitHub Terminal Commands

Git is Version Control Software for Tracking Changes

GitHub is a Website where Git "repositories" are stored.

VSCode allows you to do all git/github commands by clicking buttons, but a professional web dev must know the terminal commands for same. In fact, prior to Microsoft's aquisition of both GitHub and VSCode, developers had no alternative but to use the terminal for git.

git init makes a git repo in the current folder (working directory)

ls shows contents of working directory, but does not show hidden items, such as the .git folder, which gets made when you type git init

ls -s shows all contents of working directory, including hidden .git folder

git status shows all staged files (files ready to be committed)

git add file-name.ext adds file-name.ext to staging area, ready to be committed

git add * adds all files/folders to staging area, ready to be committed

git commit -m "first commit" commits the staged files and includes commit message

git log to get details of the commits, including messages and unique id's.

If you add more files to working directory and type git status, they will appear as untracked.

git checkout notes.txt reverts to the last committed version of notes.txt
This is the ultimate "do over" move for when you irrevocably mess up a file and just want to go back to how it was before.
