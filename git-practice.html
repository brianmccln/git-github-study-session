Git / GitHub Terminal Commands

Git is Version Control Software for Tracking Changes

Setting up a local Git repository ("repo")

GitHub is a Website where Git "repositories" are stored.

VSCode allows you to do all git/github commands by clicking buttons, but a professional web dev must know the terminal commands for same. In fact, prior to Microsoft's aquisition of both GitHub and VSCode, developers had no alternative but to use the terminal for git.

git init makes a git repo in the current folder (working directory)

ls shows contents of working directory, but does not show hidden items, such as the .git folder, which gets made when you type git init

ls -a shows all contents of working directory, including hidden .git folder

ls -R show contents of all files, including those in all subfolders--but not hidden files/folders.

ls -a -R show contents of all files, including hidden file and files in all subfolders

git status shows all staged files (files ready to be committed)

git add file-name.ext adds file-name.ext to staging area, ready to be committed

git add . adds all files/folders to staging area, ready to be committed

git add * does the same as git add ., but without including hidden files or folders, which start with a dot.

git rm --cached . -r removes all items from staging area. -r means "recursively", which means that all items in subfolders are also removed. Then type git status to confirm that the files have been unstaged.

git commit -m "first commit" commits the staged files and includes commit message

git log to get details of the commits, including messages and unique id's.

If you add more files to working directory and type git status, they will appear as untracked.

git checkout notes.txt reverts to the last committed version of notes.txt
This is the ultimate "do over" move for when you irrevocably mess up a file and just want to go back to how it was before.

Pushing a local git repo to github remote repo

These next steps require you to have first logging into github.com, making a new repo there, and copying the url.

git remote add origin https:github.com/MyGithubAcct/MyProjectFolder

git branch -M main makes a main branch, providing a destination to push to.

git push -u origin main pushes the local git repo to the remote github repo's main branch.

-u flag creates a tracking reference for every branch that you successfully push onto the remote repository. The local branch you push is automatically linked with the remote branch. This allows you to use commands such as git pull without any arguments.

touch .gitignore makes an invisible file for specifying files for git to ignore. These would be files with sensitive information, such as passwords, or .DS_Store file. The file contains information about which files are in the folder and contains options such as what icons look like. For cybersecurity reasons, .DS_Store should not be uploaded to github.

ls -a to see invisible files and folders--including .gitignore

open .gitignore to open the .gitignore file for editing in VSCode.

In .gitignore file, type the names of the file(s) to be ignored by git, one file name per line.
